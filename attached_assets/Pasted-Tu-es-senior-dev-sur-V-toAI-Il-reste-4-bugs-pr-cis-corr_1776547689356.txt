### CORRECTIF 1 — CRITIQUE : Route POST /api/consultations/:id/actes manquante

Le modal "Ajouter un acte" s'ouvre correctement mais au submit, AUCUN appel API n'est déclenché. La route backend n'existe pas.

**Fix requis :**

1. Côté serveur, ajouter la route :
```typescript
router.post('/api/consultations/:id/actes', requireAuth(), async (req, res) => {
  const { id } = req.params;
  const { description, quantite, prixUnitaireHt, tauxTva } = req.body;
  
  if (!description || !prixUnitaireHt) {
    return res.status(400).json({ error: 'Description et prix requis' });
  }
  
  const qte = Number(quantite) || 1;
  const prixHt = Number(prixUnitaireHt);
  const tva = Number(tauxTva) || 20;
  const total = qte * prixHt * (1 + tva / 100);
  
  const [acte] = await db.insert(actesTable).values({
    consultationId: Number(id),
    description,
    quantite: qte,
    prixUnitaireHt: prixHt,
    tauxTva: tva,
    total,
    clinicId: req.auth.orgId ?? req.auth.userId
  }).returning();
  
  res.json(acte);
});
```

2. Côté client, dans le composant AjouterActeModal (ou équivalent), vérifier que le `onSubmit` appelle bien :

### CORRECTIF 2 : "Qté: null comprimé" sur ordonnance AULIT

Dans `/ordonnances/1/imprimer`, la ligne Ypozane affiche "Qté: null comprimé".

**Fix requis** : dans le helper qui formate les lignes d'ordonnance, filtrer les valeurs nulles :
```typescript
const parts = [
  ligne.nomMedicament,
  ligne.dosage || null,
  ligne.voieAdmin || null,
  ligne.frequence || null,
  ligne.duree ? `pendant ${ligne.duree}` : null,
  ligne.quantite != null ? `Qté: ${ligne.quantite} ${ligne.unite ?? 'comprimé(s)'}` : null,
].filter(Boolean);
return parts.join(' — ');
```
