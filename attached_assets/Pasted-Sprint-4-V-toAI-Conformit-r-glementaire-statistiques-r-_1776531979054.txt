### BLOC A : Statistiques — calcul CA réel

Suite à la correction des factures (Sprint 2+3), les statistiques devraient fonctionner.
Vérifier et si nécessaire corriger les requêtes :

**A1 — CA aujourd'hui**
```sql
SELECT COALESCE(SUM(montant_ttc), 0) as ca_today
FROM factures 
WHERE statut = 'payee' 
AND DATE(updated_at AT TIME ZONE 'Europe/Paris') = CURRENT_DATE AT TIME ZONE 'Europe/Paris'
AND clinic_id = $1
```

**A2 — Taux d'encaissement**
```sql
SELECT 
  COUNT(*) FILTER (WHERE statut = 'payee') * 100.0 / NULLIF(COUNT(*), 0) as taux
FROM factures
WHERE clinic_id = $1 AND EXTRACT(MONTH FROM created_at) = EXTRACT(MONTH FROM NOW())
```

**A3 — Top actes & produits** 
```sql
SELECT lf.description, COUNT(*) as nb, SUM(lf.total) as ca
FROM lignes_facture lf
JOIN factures f ON f.id = lf.facture_id
WHERE f.statut = 'payee' AND f.clinic_id = $1
GROUP BY lf.description
ORDER BY ca DESC LIMIT 10
```

### BLOC E : Sécurité — validations serveur manquantes

**E1 — Toutes les routes API doivent vérifier `clinic_id`**
Pattern à appliquer partout :
```typescript
// Middleware à ajouter sur toutes les routes GET/POST/PATCH/DELETE
const ensureClinicId = (req, res, next) => {
  if (!req.auth?.orgId && !req.auth?.userId) {
    return res.status(401).json({ error: 'Non authentifié' })
  }
  req.clinicId = req.auth.orgId ?? req.auth.userId
  next()
}
```

**E2 — Rate limiting sur les routes IA (coût Anthropic)**
```typescript
import rateLimit from 'express-rate-limit'

app.use('/api/certificats/generate', aiLimiter)
```
